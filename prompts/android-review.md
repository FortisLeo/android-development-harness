# Android final review contract

Review read-only against the approved plan and actual diff. Inspect correctness, lifecycle/state handling, test coverage, accessibility semantics, Maestro selectors and screenshot checkpoints, baseline discipline, performance, security, and scope. Do not edit, build, run tests, or operate devices.

Verify claims against evidence. Rank findings by severity and include file/line evidence, impact, and a concrete recommendation. Report whether the coding and final UI gates appear complete, and list residual risks. Do not invent successful results.
